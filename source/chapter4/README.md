## 本章目标

本章的目标是实现虚拟内存机制，为每个应用程序提供独立的地址空间，实现内存隔离和保护。

## 主要内容

1. **SV39 分页机制**：实现 RISC-V 64 位架构的 39 位虚拟地址分页模式
2. **内核堆分配器**：使用 `kernel-alloc` 提供动态内存分配能力
3. **地址空间抽象**：通过 `AddressSpace` 管理虚拟地址空间
4. **内核地址空间**：为内核创建使用恒等映射的地址空间
5. **应用地址空间**：通过解析 ELF 文件为应用程序创建独立的地址空间
6. **跨地址空间切换**：使用 `ForeignContext` 和传送门机制实现 Trap 时的地址空间切换

## 代码位置

本章的代码位于 `rCore-Tutorial-in-single-workspace/ch4/` 目录下。

主要文件：
- `src/main.rs` - 内核主函数，包含地址空间创建、调度循环和系统调用实现
- `src/process.rs` - 进程管理，包含 ELF 解析和进程创建

主要依赖库：
- `kernel-vm` - 虚拟内存管理库（`AddressSpace`、`PageManager`）
- `kernel-context` - 内核上下文管理库（`ForeignContext`、`MultislotPortal`）
- `kernel-alloc` - 内核堆分配器
- `page_table` - 页表相关的数据结构
- `linker` - 链接脚本和应用程序管理
- `syscall` - 系统调用处理
- `xmas-elf` - ELF 文件解析

## 关键概念

| 概念 | 说明 |
|------|------|
| 虚拟地址空间 | 每个进程拥有独立的虚拟地址空间 |
| SV39 分页模式 | RISC-V 64 位架构的 39 位虚拟地址分页模式 |
| 页表 | 虚拟地址到物理地址的映射关系表 |
| 恒等映射 | 虚拟地址等于物理地址的映射方式 |
| ForeignContext | 跨地址空间的上下文，包含 satp |
| 传送门 (Portal) | 用于在地址空间切换时执行代码的特殊机制 |
| 地址翻译 | 内核访问用户数据时需要手动查页表翻译地址 |

## 与第三章的对比

| 特性 | 第三章 | 第四章 |
|------|--------|--------|
| 地址空间 | 所有任务共享物理地址空间 | 每个进程拥有独立的虚拟地址空间 |
| 数据结构 | `TaskControlBlock` | `Process` (含 `AddressSpace`) |
| 上下文 | `LocalContext` | `ForeignContext` |
| 内存保护 | 无 | 通过页表实现 |
| 地址转换 | 直接使用物理地址 | 通过 MMU 进行地址转换 |
| 应用加载 | 二进制数据直接执行 | 解析 ELF 文件建立映射 |
| 系统调用 | 直接访问用户指针 | 需要地址翻译 |
| 隔离性 | 弱 | 强，完全隔离 |
| 安全性 | 低 | 高，可防范 Meltdown 等漏洞 |
